# memU เพื่อสังคมไทย - วิสัยทัศน์และแนวทาง

## วิสัยทัศน์: AI เข้าถึงได้สำหรับทุกคน ไม่ใช่แค่นักพัฒนา

เราเชื่อว่า AI ที่มีความสามารถในการจดจำควรเป็นสิทธิขั้นพื้นฐานของทุกชุมชน ไม่ใช่สิทธิพิเศษของบริษัทเทคโนโลยี

**หลักการสำคัญ:**
- **เข้าถึงง่าย** — ติดตั้งได้ด้วยคำสั่งเดียว ไม่ต้องมีพื้นฐาน DevOps
- **ใช้ภาษาไทยได้** — ถามและบันทึกเป็นภาษาไทยโดยตรง
- **ปลอดภัยและโปร่งใส** — ข้อมูลอยู่ในการควบคุมของชุมชน
- **ต่อยอดได้** — เชื่อมต่อกับ LINE, N8N และระบบที่มีอยู่แล้ว

## คู่มือการติดตั้งสำหรับองค์กรชุมชน โรงเรียน และอาสาสมัคร

### ขั้นตอนที่ 1: เตรียมสภาพแวดล้อม

```bash
# ติดตั้ง Python (ต้องการเวอร์ชัน 3.10 ขึ้นไป)
python --version

# สร้าง virtual environment
python -m venv venv
source venv/bin/activate  # macOS/Linux
# หรือ venv\Scripts\activate  # Windows
```

### ขั้นตอนที่ 2: ติดตั้ง memU

```bash
pip install memu-py
```

### ขั้นตอนที่ 3: ตั้งค่า API Key

```bash
# ขอ API Key ได้ที่ platform.openai.com
export OPENAI_API_KEY=sk-...your-key-here...
```

### ขั้นตอนที่ 4: รันตัวอย่าง

```bash
python examples/community/01_teacher_student.py
```

### ตัวเลือกสำหรับการใช้งานจริง (Production)

สำหรับองค์กรที่ต้องการเก็บข้อมูลถาวร แนะนำใช้ฐานข้อมูล SQLite หรือ PostgreSQL:

```python
from memu.app import MemoryService

service = MemoryService(
    llm_profiles={"default": {"api_key": api_key, "chat_model": "gpt-4o-mini"}},
    database_config={
        "metadata_store": {"provider": "sqlite", "url": "sqlite:///community_memory.db"}
    },
)
```

## แนวปฏิบัติด้านความเป็นส่วนตัวและจริยธรรม

### หลักการพื้นฐาน

1. **ขอความยินยอมก่อน** — ก่อนบันทึกข้อมูลของบุคคลใด ต้องได้รับการยินยอมอย่างชัดเจน
2. **บอกวัตถุประสงค์** — แจ้งให้บุคคลนั้นรู้ว่าข้อมูลจะถูกใช้เพื่ออะไร
3. **เข้าถึงเท่าที่จำเป็น** — จำกัดการเข้าถึงข้อมูลเฉพาะผู้ที่มีความจำเป็น
4. **ลบเมื่อหมดความจำเป็น** — กำหนดระยะเวลาการเก็บข้อมูลและลบเมื่อครบกำหนด

### แนวปฏิบัติพิเศษสำหรับข้อมูลเด็ก

- **ห้ามบันทึกข้อมูลที่ระบุตัวตนเด็กได้โดยตรง** โดยไม่ได้รับความยินยอมจากผู้ปกครอง
- ใช้รหัสนักเรียนแทนชื่อจริงในระบบ
- เก็บข้อมูลเฉพาะที่จำเป็นสำหรับการพัฒนาการเรียนรู้
- ผู้ปกครองมีสิทธิ์ขอดูและลบข้อมูลของบุตรได้ตลอดเวลา

### แนวปฏิบัติพิเศษสำหรับข้อมูลผู้สูงอายุ

- ข้อมูลสุขภาพถือเป็นข้อมูลอ่อนไหวตาม PDPA
- ต้องได้รับความยินยอมจากผู้สูงอายุหรือผู้ดูแลทางกฎหมาย
- เก็บเฉพาะข้อมูลที่ช่วยในการดูแลสุขภาพโดยตรง
- มีระบบแจ้งเตือนเมื่อข้อมูลถูกเข้าถึงหรือแก้ไข

### การปฏิบัติตาม PDPA (พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562)

```python
# ตัวอย่างการบันทึกการยินยอม
consent_record = {
    "user_id": "รหัสผู้ใช้",
    "consent_date": "2025-02-01",
    "purpose": "การดูแลสุขภาพและการนัดหมอ",
    "consent_given": True,
    "can_withdraw": True,  # ผู้ใช้ถอนความยินยอมได้เสมอ
}
```

## ข้อมูลสำหรับนักพัฒนาที่ต้องการต่อยอด

### สถาปัตยกรรมแนะนำสำหรับชุมชน

```
ผู้ใช้ (LINE / Web / Mobile App)
    ↕
API Gateway (FastAPI / Flask)
    ↕
memU MemoryService
    ↕
ฐานข้อมูล (SQLite สำหรับขนาดเล็ก / PostgreSQL สำหรับขนาดใหญ่)
```

### ตัวอย่างการเชื่อมต่อ LINE Messaging API

```python
from linebot.v3.messaging import MessagingApi, ReplyMessageRequest, TextMessage
from memu.app import MemoryService

service = MemoryService(
    llm_profiles={"default": {"api_key": openai_key, "chat_model": "gpt-4o-mini"}}
)

async def handle_message(user_id: str, text: str) -> str:
    # บันทึกข้อความของผู้ใช้
    # (ถ้าต้องการ)

    # สืบค้นจากความจำ
    result = await service.retrieve(
        queries=[{"role": "user", "content": text}],
        where={"user_id": user_id},
    )

    items = result.get("items", [])
    if items:
        return items[0].get("summary", "ไม่พบข้อมูล")
    return "ไม่พบข้อมูลที่เกี่ยวข้อง"
```
